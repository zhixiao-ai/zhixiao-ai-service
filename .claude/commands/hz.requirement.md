# hz.requirement.md

该命令需要实现如下内容
1. 根据用户的需求细化或者补充需求文档，细化内容参考 **需求内容**内容；
2. 对于不确定的内容可以请用户回答一下；
3. 该文档生成的内容需要围绕需求本身，**不需要考虑架构设计、模型设计、接口设计**，最重要的表明需求；
4. 需求文档需要存储到docs/功能名/功能名_requirements.md；


需求内容：
# xx功能需求
1. **需求背景和概述**
2. **影响用户、角色、重要程度**

## 功能性描述
### 功能点1：xxx
### 功能点2：xxx
### 功能点x：xxx

## 非功能性描述
1. **运行环境**： 操作系统、数据库、中间件等；
2. **性能要求**；
